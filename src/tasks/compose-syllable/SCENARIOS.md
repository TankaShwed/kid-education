# Сценарии тестов: «Собери слог»

Описание в терминах предметной области и элементов интерфейса. В тестах привязка — по кнопке «Начать», по слотам для слога и по пулу букв; озвучку проверяем через мок TTS.

**Элементы интерфейса:** кнопка «Начать», слоты для слога (места под буквы), пул букв (буквы для перетаскивания), подсказки «Слушай задание…» и «Перетащи буквы в слоты по порядку».

## Сценарий 1: Успешный путь

1. На экране раунд «Собери слог НО»: видна только кнопка «Начать».
2. Пользователь нажимает «Начать».
3. Озвучивается инструкция «Собери слог» и целевой слог; на экране появляются слоты для слога и пул букв.
4. Пользователь перетаскивает буквы в слоты в правильном порядке (сначала Н, потом О).
5. Озвучивается «Правильно! Молодец!», задание переходит к следующему раунду.

## Сценарий 2: Неверный слог и повтор

1. На экране раунд «Собери слог НО», пользователь нажимает «Начать»; появляются слоты и пул букв.
2. Пользователь перетаскивает буквы в неправильном порядке (сначала О, потом Н).
3. Озвучивается подсказка («Это слог он. Попробуй ещё раз.»); слоты очищаются, буквы возвращаются в пул.
4. Пользователь перетаскивает буквы в правильном порядке (Н, затем О).
5. Озвучивается «Правильно! Молодец!».

---

**Привязка в тестах (data-testid):** кнопка «Начать» — `compose-syllable-start`, слоты — `compose-syllable-slots`, пул букв — `compose-syllable-pool`. Буквы и слоты — по тексту/роли или по тестовым идентификаторам, если добавлены.
