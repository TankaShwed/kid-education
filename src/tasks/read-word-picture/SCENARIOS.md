# Сценарии тестов: «Прочитай слово и выбери картинку»

Описание в терминах предметной области и элементов интерфейса. В тестах привязка по data-testid; озвучку проверяем через мок TTS.

**Элементы интерфейса:** кнопка «Начать», слово по слогам (каждый слог кликабелен), варианты картинок (кнопки с alt), подсказка «Слушай задание…».

## data-testid

- `read-word-picture-round` — корневой контейнер раунда
- `read-word-picture-start` — кнопка «Начать»
- `read-word-picture-word` — блок со словом по слогам
- `read-word-picture-syllable-{index}` — кнопка слога (по клику озвучивается только этот слог)
- `read-word-picture-options` — контейнер вариантов картинок
- `read-word-picture-option-{id}` — кнопка варианта (id из раунда)
- `read-word-picture-hint` — подсказка «Слушай задание…»

## Сценарий: успешный путь

Раунд: слово МАМА, варианты с correctId «mama».

1. Пользователь видит кнопку «Начать», нажимает её.
2. Озвучивается инструкция «Прочитай слово и выбери картинку», появляются слово по слогам и варианты.
3. Пользователь нажимает на слог — озвучивается только этот слог.
4. Пользователь выбирает правильную картинку (mama) — озвучивается «Правильно», запрашивается следующий раунд.

## Сценарий: неправильный выбор

1. После старта пользователь выбирает неправильный вариант.
2. Озвучивается фидбек («Нет. Выбери картинку к слову.» + выбранный alt), выбранный вариант исчезает.
3. Пользователь выбирает правильный вариант — «Правильно», следующий раунд.
